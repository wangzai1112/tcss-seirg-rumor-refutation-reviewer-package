# 外部验证数据集选择 v3

## 结论

新增外部验证优先采用 Mendeley 的 COVID-19 Rumor Detection dataset。它比 ESOC 更适合作为传播曲线验证，因为它含有发布时间字段和谣言/非谣言标签，可以按天或小时构造真实事件的传播代理曲线。CHECKED 继续作为中文语境校准数据，新增数据只用于外部验证，避免同一数据既校准又验证。

## 推荐数据集 1：Mendeley COVID-19 Rumor Detection dataset

来源：https://data.mendeley.com/datasets/xcz8vb448y/1

适合原因：

- 有 `created_at` 字段，可构造时间序列。
- 有谣言/非谣言标签，可区分传播对象。
- 有文本与互动字段，可形成传播强度代理指标。
- COVID-19 主题与政府干预、公共信息治理语境一致。

使用方式：

1. 下载数据后只做字段清洗、时间聚合和尺度对齐。
2. 不重新调传播率、干预强度等核心参数；如果必须调整，只允许做时间尺度或幅度尺度匹配。
3. 用模型输出的总传播者曲线或 I1 曲线与真实谣言曲线比较峰值位置、曲线形状和累计负荷方向。
4. 论文中写成“外部趋势验证”，不要写成“精确拟合真实传播链”。

## 推荐数据集 2：PHEME rumour datasets

来源：

- https://figshare.com/articles/dataset/PHEME_dataset_of_rumours_and_non-rumours/4010619
- https://figshare.com/articles/dataset/PHEME_dataset_for_Rumour_Detection_and_Veracity_Classification/6392078

适合原因：

- 按突发新闻事件组织。
- 包含谣言与非谣言传播记录。
- 常用于谣言检测和传播研究，学术接受度较高。

限制：

- Twitter 历史数据可能存在删除、缺失或再水化问题。
- 与中国语境和政府干预变量不完全一致。

建议作为备用外部验证或补充泛化验证。

## 辅助数据集：ESOC COVID-19 misinformation dataset

来源：https://esoc.princeton.edu/publications/esoc-covid-19-misinformation-dataset

ESOC 更适合作为事实核查、政策语境和谣言主题背景，不一定适合直接构造传播时间曲线。如果使用 ESOC，建议把它放在数据支撑或讨论部分，而不是作为主外部验证曲线。

## 与论文的衔接写法

第五章可以新增一个小节：

“为避免模型只在 CHECKED 数据上表现较好，本文进一步选择一个独立公开数据集进行外部趋势验证。验证过程中不重新估计核心传播参数，仅进行时间尺度和幅度尺度对齐。比较指标包括峰值出现顺序、峰值时间误差、RMSE、MAE 和累计传播负荷差异。”

如果新增数据的趋势验证不理想，也不应隐藏。可以写成：

“外部数据验证显示，模型可以较好捕捉传播曲线的单峰或多峰变化方向，但对具体达峰时间仍存在偏差。这说明模型更适合作为机制分析工具，而不是在缺少完整转发链数据时进行精确预测。”
